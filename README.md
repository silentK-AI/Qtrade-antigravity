# 量化交易工具 - 基于CrewAI的多智能体股票价格预测系统

这是一个针对中国股市的量化交易工具，使用CrewAI多智能体框架构建，通过四个专业智能体协作完成股票价格预测任务。

## 功能特点

- **多智能体协作**：四个专业智能体分工协作，完成从数据收集到预测评估的完整流程
- **全面数据收集**：自动获取股票历史交易数据和最新相关资讯
- **智能分析预测**：基于历史数据和资讯，预测股票当日最高价和最低价
- **结果评估优化**：对预测结果进行评估，提供优化建议
- **智能错误处理**：当主模型遇到 429 错误（API 速率限制）时，自动切换到备用模型 `gemini-2.0-flash-live`

## 系统架构

### 四个智能体

1. **数据收集智能体（Data Collector Agent）**
   - 职责：获取股票历史交易数据
   - 数据范围：过去三年的每日交易数据
   - 包含指标：开盘价、收盘价、最高价、最低价、成交量、量比、换手率等

2. **资讯收集智能体（News Collector Agent）**
   - 职责：搜索并整理股票相关资讯
   - 资讯类型：
     - 公司公告
     - 券商研报
     - 大宗商品价格
     - 美联储议息决议
     - 美国关税政策

3. **分析决策智能体（Analysis Agent）**
   - 职责：基于数据和资讯进行价格预测
   - 分析方法：
     - 技术分析（趋势、支撑位/阻力位、技术指标）
     - 量价关系分析
     - 基本面分析
     - 市场环境分析
   - 输出：当日最高价和最低价预测

4. **评估智能体（Evaluator Agent）**
   - 职责：评估预测结果并提供优化建议
   - 评估维度：
     - 预测合理性
     - 风险识别
     - 问题诊断
     - 优化建议

## 安装步骤

### 1. 克隆或下载项目

```bash
cd Quati-Trade
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置环境变量

创建 `.env` 文件（可选，API密钥已在config.py中配置）：

```bash
GEMINI_API_KEY=your_gemini_api_key_here
GEMINI_MODEL=gemini-2.0-flash
```

注意：API密钥已在 `config.py` 中默认配置，如需修改可通过环境变量覆盖。

### 4. 验证安装

确保所有依赖包已正确安装，特别是：
- `crewai[google-genai]`: CrewAI框架（包含Google Gemini支持）
- `akshare`: 中国股市数据获取
- `litellm`: LiteLLM支持（CrewAI需要）

## 使用方法

### 基本用法

```bash
python main.py --stock-code 000001
```

### 指定日期范围

```bash
python main.py --stock-code 000001 --start-date 20210101 --end-date 20231231
```

### 保存结果到文件

```bash
python main.py --stock-code 000001 --output result.json
```

### 参数说明

- `--stock-code`: 股票代码（必需），如：000001（平安银行）
- `--start-date`: 开始日期（可选），格式：YYYYMMDD，默认三年前
- `--end-date`: 结束日期（可选），格式：YYYYMMDD，默认今天
- `--output`: 输出文件路径（可选），默认打印到控制台

## 工作流程

1. **数据收集阶段**
   - 数据收集智能体获取股票历史交易数据
   - 资讯收集智能体搜索相关资讯
   - 两个智能体可以并行工作

2. **分析预测阶段**
   - 分析决策智能体接收数据和资讯
   - 运用多种分析方法进行综合预测
   - 输出当日最高价和最低价预测

3. **评估优化阶段**
   - 评估智能体对预测结果进行评估
   - 识别潜在问题和风险
   - 提供具体的优化建议

## 数据源

- **股票数据**：使用 `akshare` 获取中国股市数据
- **资讯数据**：通过 `akshare` 和相关API获取各类资讯

## 注意事项

1. **API密钥**：已配置Google Gemini API密钥，如需修改可在config.py或.env文件中设置
2. **模型配置**：
   - 主模型：`gemini-2.0-flash`（默认）
   - 备用模型：`gemini-2.0-flash-live`（当主模型遇到 429 错误时自动切换）
   - 可通过环境变量 `GEMINI_MODEL` 和 `GEMINI_FALLBACK_MODEL` 自定义
3. **错误处理**：系统会自动检测 429 错误（API 速率限制），并自动切换到备用模型，无需手动干预
4. **数据获取**：某些数据源可能需要网络连接或API访问权限
5. **预测准确性**：预测结果仅供参考，不构成投资建议
6. **风险提示**：股市有风险，投资需谨慎

## 常见问题

### Q: 如何获取股票代码？

A: 中国A股股票代码为6位数字，如：
- 000001: 平安银行
- 600000: 浦发银行
- 000002: 万科A

### Q: 数据获取失败怎么办？

A: 检查网络连接，确保可以访问数据源。某些数据可能需要特定的访问权限。

### Q: 如何提高预测准确性？

A: 
- 确保数据质量完整
- 收集更多相关资讯
- 优化分析模型参数
- 参考评估智能体的优化建议
