# 系统架构说明

## 整体架构

本系统采用CrewAI多智能体框架，通过四个专业智能体协作完成股票价格预测任务。

```
┌─────────────────────────────────────────────────────────────┐
│                     量化交易工具系统                          │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────────────────────────┐
        │          CrewAI Crew 协调器           │
        └───────────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
        ▼                   ▼                   ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  数据收集    │    │  资讯收集    │    │  分析决策    │
│   智能体     │    │   智能体     │    │   智能体     │
└──────────────┘    └──────────────┘    └──────────────┘
        │                   │                   │
        └───────────────────┼───────────────────┘
                            │
                            ▼
                    ┌──────────────┐
                    │   评估智能体  │
                    └──────────────┘
```

## 工作流程

### 阶段1: 数据收集（并行执行）

1. **数据收集智能体**
   - 使用 `akshare` 获取股票历史交易数据
   - 数据范围：过去三年（或指定日期范围）
   - 包含指标：开盘价、收盘价、最高价、最低价、成交量、量比、换手率等

2. **资讯收集智能体**
   - 搜索公司公告
   - 搜索券商研报
   - 获取大宗商品价格
   - 获取美联储议息决议
   - 获取美国关税政策

### 阶段2: 分析预测

3. **分析决策智能体**
   - 接收数据收集和资讯收集的结果
   - 运用技术分析、基本面分析等方法
   - 预测当日最高价和最低价
   - 评估预测置信度

### 阶段3: 评估优化

4. **评估智能体**
   - 评估预测结果的合理性
   - 识别潜在问题和风险
   - 提供优化建议

## 智能体详细说明

### 1. 数据收集智能体 (Data Collector Agent)

**角色**: 股票数据收集专家

**职责**:
- 获取股票历史交易数据
- 确保数据完整性和准确性
- 提供数据统计摘要

**工具**:
- `get_stock_history_tool`: 获取股票历史数据

**输出**:
- 股票历史数据报告
- 数据统计摘要
- 最近交易日数据

### 2. 资讯收集智能体 (News Collector Agent)

**角色**: 股票资讯收集专家

**职责**:
- 搜索并整理股票相关资讯
- 分类和重要性评估
- 分析资讯对价格的影响

**工具**:
- `search_stock_news_tool`: 搜索股票相关资讯

**输出**:
- 各类资讯汇总
- 重要资讯详细内容
- 资讯影响分析

### 3. 分析决策智能体 (Analysis Agent)

**角色**: 股票价格分析专家

**职责**:
- 综合分析历史数据和资讯
- 运用多种分析方法
- 预测当日最高价和最低价

**分析方法**:
- 技术分析（趋势、支撑位/阻力位、技术指标）
- 量价关系分析
- 基本面分析
- 市场环境分析

**输出**:
- 当日最高价预测
- 当日最低价预测
- 预测依据和逻辑
- 置信度评估

### 4. 评估智能体 (Evaluator Agent)

**角色**: 交易策略评估专家

**职责**:
- 评估预测结果
- 识别问题和风险
- 提供优化建议

**评估维度**:
- 预测合理性
- 逻辑可靠性
- 风险识别
- 改进建议

**输出**:
- 评估报告
- 问题识别
- 优化建议

## 任务依赖关系

```
数据收集任务 ──┐
              ├──> 分析任务 ──> 评估任务
资讯收集任务 ──┘
```

- 数据收集和资讯收集可以并行执行
- 分析任务需要等待数据和资讯收集完成
- 评估任务需要等待分析任务完成

## 数据流

```
输入: 股票代码
  │
  ├─> 数据收集智能体 ──> 历史交易数据
  │
  ├─> 资讯收集智能体 ──> 相关资讯
  │
  └─> 分析决策智能体 ──> 价格预测
        │
        └─> 评估智能体 ──> 评估报告
              │
              └─> 最终输出
```

## 技术栈

- **框架**: CrewAI
- **LLM**: Google Gemini Pro
- **数据源**: akshare (中国股市数据)
- **数据处理**: pandas, numpy
- **工具**: crewai-tools

## 扩展点

### 添加新的数据源

1. 在 `tools/` 目录下创建新的工具文件
2. 使用 `@tool` 装饰器定义工具
3. 在相应的智能体中添加工具

### 自定义分析模型

1. 在 `agents/analysis_agent.py` 中修改分析逻辑
2. 或创建新的分析工具
3. 在分析智能体中添加工具

### 优化评估标准

1. 在 `agents/evaluator_agent.py` 中调整评估逻辑
2. 添加新的评估维度
3. 优化建议生成逻辑

## 性能优化

- 数据收集和资讯收集并行执行，提高效率
- 使用记忆功能，智能体可以记住之前的对话
- 设置合理的超时和迭代限制
- 控制API请求频率（max_rpm）

## 注意事项

1. **API密钥**: 已配置Google Gemini API密钥，可在config.py或.env文件中修改
2. **数据获取**: 某些数据源可能需要网络连接
3. **预测准确性**: 预测结果仅供参考，不构成投资建议
4. **风险提示**: 股市有风险，投资需谨慎
